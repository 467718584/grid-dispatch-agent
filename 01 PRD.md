# Grid Dispatch Agent - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
**Grid Dispatch Agent (GDA)** - 电网调度智能Agent

### 1.2 产品定位
一个轻量化的通用Agent框架，用于电网调度业务。框架不绑定具体业务，通过即插即用的Skill机制实现业务定制。

### 1.3 核心价值
| 价值 | 说明 |
|------|------|
| **通用性** | 不绑定具体业务，通过Skill扩展适应不同流程 |
| **轻量化** | 仅依赖Python标准库 + OpenAI兼容API |
| **可复用** | Skill低耦合，业务切换只需更换Skill配置 |
| **易迁移** | 更换系统/现场时，有LLM即可运行 |

### 1.4 目标用户
- 电网调度工程师
- 电力系统运维人员
- 业务流程定制开发者

---

## 2. 功能需求

### 2.1 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| F1: Skill注册机制 | 支持动态注册业务Skill，Skill包含：取数、计算、推断、格式化等 | P0 |
| F2: 流程编排引擎 | 根据Task自动编排执行顺序，支持顺序/条件/并行 | P0 |
| F3: LLM调用接口 | 支持OpenAI兼容API（局域网本地部署） | P0 |
| F4: 结构化输出 | 输出JSON格式，支持key定制 | P0 |
| F5: Skill即插即用 | 运行时动态加载Skill，替换业务无需重启 | P1 |
| F6: 多Skill组合 | 单一业务流程可由多个Skill组合完成 | P1 |

### 2.2 非功能性需求

| 需求 | 要求 |
|------|------|
| N1: 部署复杂度 | 纯Python，pip install requests即可 |
| N2: 代码量 | 核心框架不超过500行 |
| N3: 响应时间 | 单次Task处理不超过30秒（不含LLM延迟） |
| N4: 扩展性 | 新增业务时无需修改框架代码 |

---

## 3. 技术约束

| 约束 | 说明 |
|------|------|
| T1: LLM | OpenAI兼容API（支持局域网URL） |
| T2: 数据库 | 预留接口，暂不实现具体对接 |
| T3: 输出 | JSON with Keys |
| T4: Python | 3.8+ |
| T5: 依赖 | requests (仅HTTP调用) |

---

## 4. 范围定义

### 4.1 第一期范围（本期）
- 通用Agent框架开发
- 3个示例Skill（模拟取数、计算、格式化）
- Skill注册与编排机制
- JSON输出

### 4.2 第二期范围（后续）
- 真实数据库接口对接
- 真实业务Skill封装
- 输出格式扩展

---

## 5. 风险与假设

### 5.1 假设
- 用户有Python 3.8+环境
- LLM已部署并可通过HTTP访问
- 用户可提供业务Skill的输入输出规范

### 5.2 风险
- 数据库接口文件暂未提供，需预留接口
- Skill编排逻辑可能需要根据业务调整

---

## 6. 验收标准

| 标准 | 说明 |
|------|------|
| AC1 | Agent可启动并响应Task |
| AC2 | 可注册自定义Skill |
| AC3 | Skill可自由组合执行 |
| AC4 | 输出为JSON格式 |
| AC5 | 更换Skill时无需修改框架代码 |
